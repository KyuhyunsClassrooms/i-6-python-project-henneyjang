# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 퀴즈를 접목한 JRPG 형식의 턴제 게임



## 2. 문제 상황

흥미롭게 플레이할 수 있는 턴제 게임 프로그램 제작



## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 플레이어의 이름 (문자열) 
- 퀴즈의 정답 (문자열)
- 플레이어의 행동

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 사용자가 입력한 문제의 정답이 정확한지 아닌지를 if else문으로 판단한다.
- 2차원 리스트를 활용해 스테이지별 몬스터 정보(이름, 체력, 공격력)와 퀴즈 데이터(질문, 정답)를 구조화하여 관리한다
- 

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 현재 라운드 정보 및 플레이어/몬스터의 남은 체력(HP) 상태
- 퀴즈 질문 문항
-전투 승리 또는 패배 결과 문구
---

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:
몬스터 데이터, 퀴즈 데이터


### 한 행의 의미

한 행(Row)은 하나의 몬스터 정보를 나타낸다.
 
### 각 열의 의미

[0번 열: 몬스터 이름(str), 1번 열: 체력(int), 2번 열: 공격력(int)]`

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획: 사용자의 입력이 정답인지 판단한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획: 몬스터나 사용자의 HP가 0이 될 때까지 반복한다.
